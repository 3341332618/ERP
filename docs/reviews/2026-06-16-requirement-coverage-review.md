# ERP 需求覆盖自查

日期：2026-06-16

## 结论

当前项目不是完整覆盖需求说明书的全部 ERP 功能。已实现的是可运行的核心演示闭环：登录、角色菜单、基础资料、单据创建/提交、仓库审核、库存变动、结算生成、个人头像和商品图片上传。

需求说明书中仍有一批未覆盖或只做了简化实现的功能。后续不能把“有菜单入口”视为“业务已完成”。

## 高优先级缺口

| 模块 | 当前状态 | 缺口 | 处理建议 |
| --- | --- | --- | --- |
| 库存调拨 | 有菜单和路由，但打开 `/inventory/transfer` 时实际显示采购入库配置 | 路由没有 `type` 参数，`DocumentView` 判断不到 `transfer`，导致库存调拨入口错配 | 立即修复 |
| 库存调拨业务 | 后端有 `STOCK_TRANSFER` 枚举和库存增减逻辑 | 调出仓库、调入仓库默认同一个仓库；没有选择仓库、选择商品、调拨数量、备注的编辑流程；没有独立的调拨状态文案 | 作为下一阶段核心功能补齐 |
| 采购/销售单据 | 可创建、提交、删除、查看简化单据 | 没有真正的新增/修改表单，没有选择仓库/供应商/客户/商品窗口，没有商品明细编辑，没有关联退货单据逻辑 | 分采购入库、采购退货、销售出库、销售退货逐项补齐 |
| 库存分布 | 能展示库存余额 | 当前只显示仓库标识、商品标识和数量；没有按商品聚合、商品图片、商品名称、分类、品牌、仓库明细查看和合计 | 补库存视图模型和详情弹窗 |
| 审核 | 能审核通过/拒绝并影响库存 | 审核列表字段不足；没有完整操作记录；库存调拨没有区分调拨出库/调拨入库双侧审核 | 补审核投影和操作记录 |

## 中优先级缺口

| 模块 | 当前状态 | 缺口 |
| --- | --- | --- |
| 基础资料禁用 | 可启用/禁用 | 缺少“存在库存不可禁用”“流转中单据不可禁用”等业务限制 |
| 商品批量导入 | 未实现 | 需求要求 Excel 模板下载、xls/xlsx 上传、500KB 校验、逐行错误提示 |
| 查询与分页 | 仅有少量关键字过滤 | 缺少需求中的多条件查询、时间范围、分类/品牌下拉、分页 |
| 结算详情 | 有收入/支出列表 | 缺少查看结算单详情、查看关联单据详情、单据类型筛选 |
| 消息中心 | 有消息列表 | 缺少最多 10 条展示、展开全部、精确消息格式和调拨入/出库消息拆分 |
| 数据持久化 | 当前是内存 Store | 需求级 ERP 应落数据库、迁移脚本、事务和真实持久化 |

1. 修复库存调拨入口错配，确保 `/inventory/transfer` 使用 `stock-transfer` 配置。
2. 为后端库存调拨增加可指定调出仓库、调入仓库、商品和数量的创建能力。

4. 再补采购/销售四类单据的真实编辑明细。
