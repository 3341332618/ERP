# ERP 项目文档设计

## 目标

为当前 ERP 项目补齐两类项目文档，同时服务课程验收、项目演示和后续开发接手。

## 交付物

### `README.md`

根目录主文档，内容以可运行、可验证为准，包含：

- 项目简介与当前实现范围
- 核心功能和业务闭环
- 技术栈与目录结构
- 环境要求、安装和启动步骤
- 演示账号与访问地址
- 后端配置、API 和数据存储说明
- 测试、构建与部署命令
- 常见问题、限制和项目状态

### `docs/project-showcase.md`

面向课程答辩和现场演示，内容包含：

- 项目定位与解决的问题
- 系统角色、模块和技术亮点
- 推荐演示准备和演示流程
- 业务闭环讲解要点
- 答辩陈述参考和常见问题

## 内容原则

- 只记录仓库中已实现、可从代码或配置验证的功能。
- 所有命令使用当前 Windows PowerShell 环境可直接复制的形式。
- 对内存数据、默认 JWT 密钥等开发环境限制如实说明。
- 不修改任何业务代码、配置或用户当前未提交的更改。
- 文档统一保存为 UTF-8，修复现有 README 的中文乱码。

## 验证方式

- 对照 Maven、npm 和应用配置核对版本、端口及脚本。
- 对照路由、控制器和测试用例核对功能、API 与业务规则。
- 运行前端构建和后端测试，确认文档中的核心命令有效。
- 扫描文档中的占位符、无效链接和未经验证的描述。
