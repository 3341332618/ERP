# 双登录入口设计说明

## 背景

当前系统已经把“学员测试账号”和“学员 ERP 子账号”拆开：

- `student01` / `student02` 是学员主账号，只负责测试竞赛、提交缺陷报告和附件。
- `student01_purchase_staff`、`student01_warehouse_staff` 等是该学员专属 ERP 子账号，用于采购、库存、销售、结算业务操作。

如果只保留一个登录页，用户需要自己理解“主账号”和“子账号”的区别，容易误以为登录后还要再次登录。登录入口应在第一步就把使用场景分开。

## 目标

提供两个前端登录入口，让用户在登录前就选择正确场景：

1. ERP 端登录：面向管理员、企业岗位账号、学员 ERP 子账号。
2. 学员测试端登录：面向学员主账号。

两个入口仍共用后端 `/auth/login` 接口，不新增第二套认证系统，不增加后端登录状态复杂度。

## 非目标

- 不做“登录后再次登录”。
- 不新增会话切换器或模拟登录。
- 不改变现有 JWT、角色、菜单、数据隔离模型。
- 不把学员主账号恢复为全能 ERP 账号。

## 方案比较

### 方案 A：单登录页增加账号说明

实现最少，但用户仍需要在同一个表单里理解多种账号。学员和 ERP 子账号的边界不明显，后续演示时仍容易解释不清。

### 方案 B：两个前端登录入口，共用后端认证

新增 `/login` 和 `/student-login` 两个入口页，分别限制账号类型。后端不变，前端根据登录返回的角色和用户名判断是否允许进入当前入口。

这是推荐方案。它能解决“是不是登录后还有一个登录页”的误解，同时代码改动较小。

### 方案 C：登录后在系统内一键切换学员岗位

体验最顺，但会引入身份切换、审计归属、权限模拟等额外复杂度。当前项目优先保证业务可用和演示清晰，暂不采用。

## 选定设计

采用方案 B。

### 路由

- `/login`：ERP 端登录页，作为默认未登录入口。
- `/student-login`：学员测试端登录页。
- 两个登录页互相提供文字链接。
- 已登录用户访问任一登录页时，保持当前行为：允许展示登录页并可重新登录，或由现有逻辑按后续实现细节决定是否跳转首页。

### ERP 端登录规则

允许登录：

- `SUPER_ADMIN`
- `ADMIN`
- `PURCHASE_MANAGER`
- `PURCHASE_STAFF`
- `WAREHOUSE_MANAGER`
- `WAREHOUSE_STAFF`
- `SALES_MANAGER`
- `SALES_STAFF`
- `SETTLEMENT_MANAGER`

不允许登录：

- `STUDENT`

当 `STUDENT` 账号从 ERP 端登录时，前端在拿到登录结果后立即退出本次登录状态，并提示：“学员主账号请前往学员测试端登录”。

### 学员测试端登录规则

允许登录：

- `STUDENT`

不允许登录：

- 所有非 `STUDENT` 角色，包括学员 ERP 子账号。

当 ERP 岗位账号从学员测试端登录时，前端在拿到登录结果后立即退出本次登录状态，并提示：“ERP 岗位账号请前往 ERP 端登录”。

### 登录成功跳转

- ERP 端登录成功：跳转 `/home`，由现有菜单控制可见模块。
- 学员测试端登录成功：跳转 `/competition/reports`，直接进入缺陷报告列表/提交场景。

### 页面文案

ERP 端登录页重点说明：

- 用于管理员和 ERP 岗位账号。
- 学员 ERP 子账号也从这里登录。
- 示例账号保留管理员和常用业务账号，可补充 `student01_purchase_staff` 示例。

学员测试端登录页重点说明：

- 用于学员主账号。
- 登录后只进行缺陷发现、报告提交、附件提交。
- 示例账号使用 `student01` / `student02`。

### 组件结构

优先复用当前 `LoginView.vue` 的视觉样式和背景图。

实现时建议抽出一个可复用登录组件，例如：

- `LoginPanel.vue`：表单、快捷账号、提交按钮、错误提示。
- `LoginView.vue`：ERP 端配置。
- `StudentLoginView.vue`：学员测试端配置。

如果抽组件导致改动过大，可先保留两个页面文件，但应避免复制过多业务逻辑。

### 测试要求

前端测试至少覆盖：

- 路由存在 `/login` 和 `/student-login`。
- 未登录访问业务页仍跳转 `/login`。
- ERP 端页面包含 ERP 端文案和学生测试端链接。
- 学员测试端页面包含学员测试端文案和 ERP 端链接。
- 登录页保留当前背景图和淡蓝色登录框样式。

后端无需新增认证测试，因为后端认证接口和角色模型不变。

### 文档更新

更新 README 和操作手册：

- 明确两个入口地址。
- 明确 `student01` 登录学员测试端。
- 明确 `student01_purchase_staff` 等登录 ERP 端。
- 删除会让人误解成“登录后还要再登录”的表述。

## 验收标准

- 用户打开 `/login` 能清楚知道这是 ERP 端登录。
- 用户打开 `/student-login` 能清楚知道这是学员测试端登录。
- `student01` 不会从 ERP 端进入系统。
- `student01_purchase_staff` 不会从学员测试端进入系统。
- 学员主账号登录后直接进入测试竞赛相关页面。
- ERP 子账号登录后进入对应 ERP 菜单。
- 前端测试、前端构建通过。
